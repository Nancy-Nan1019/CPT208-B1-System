# OpenMind User Guide

## Purpose

This short guide is intended for assessors, tutors, or first-time users who want to run the system correctly and see the main discussion workflow.

## Important Requirement

To enter and demonstrate the discussion room properly, you need at least:
- `1 teacher account`
- `2 student accounts`

This is required because the system is designed with a minimum discussion-group size of `2 students`.

If only one student logs in, the teacher cannot demonstrate the full grouping and discussion process correctly.

## Recommended Setup

Use:
- one browser window for the teacher
- two additional browser windows, two different browsers, or incognito windows for the two students

This helps keep the three accounts logged in at the same time.

## Quick Demo Workflow

### 1. Teacher Login

1. Open the system login page.
2. Log in using the teacher account.
3. Enter the teacher dashboard.

### 2. Create a Session

1. Create a new discussion session.
2. Enter a topic.
3. Set the duration.
4. Set the group size.
5. Save or create the session.

### 3. Student Login and Join

1. Open two separate student browser sessions.
2. Log in with two different student accounts.
3. If required, choose personality type `E` or `I`.
4. In the waiting room, let both students join the same teacher-created session.

### 4. Start Grouping

1. Return to the teacher window.
2. Confirm that at least two students have joined.
3. Use the grouping function.
4. Start the discussion session.

### 5. Discussion Room Demo

In the student discussion room, assessors can test:
- `Hold to Speak`
- AI discussion guidance
- real-time ranking updates
- speaking log updates
- race-track or participation feedback

In the teacher session page, assessors can test:
- live group monitoring
- silent-group alerts
- individual and group analytics
- speaking timeline view

### 6. End the Session

1. End the discussion from the teacher side.
2. Open the result page.
3. Review rankings, group balance, participation results, and CSV export if needed.

## Troubleshooting Tips

- If login works in incognito mode but not in a normal browser window, clear `localStorage` and refresh.
- If a student cannot enter the discussion room, check whether at least two students joined the same session.
- If the teacher cannot proceed, check whether the session was created successfully and whether the students joined the correct room.

## Recommended Assessor Path

For the smoothest demonstration:
1. Log in as the teacher.
2. Create one session.
3. Log in as two students.
4. Join the same session with both students.
5. Start grouping and begin the discussion.
6. Test speaking, AI guide, and real-time updates.
7. End the session and open the result page.
